# Phase 3 数据源规划

## 1. 总原则

* 本阶段只规划数据源，不接入真实 API。
* 所有真实数据源接入前必须单独验证。
* 所有外部数据必须保留 source_name、source_type、url、created_at、updated_at。
* 所有事实性结论必须绑定 evidence_id。
* 数据不足时必须显示“数据不足”。
* AI 解读必须标记为“AI草稿 / 待人工确认”。
* X / KOL / 社交媒体信息只能作为线索，不能直接作为事实结论。
* 本系统不提供投资建议，不自动交易，不自动下单。

## 2. 数据源分层

### 2.1 官方披露数据

* 用途：验证公司业务、产品、收入分部、管理层表述、风险披露和监管文件。
* 可能覆盖的模块：基础投研 / 产业链图谱 / 候选股雷达模块、深度投研模块、重点跟踪池 / 量化策略标注模块。
* 数据可信度：高。
* 是否需要人工确认：需要，尤其是公司披露与主题受益之间的解释关系。
* 风险点：披露文本可能滞后，业务表述可能宽泛，不能把披露存在直接等同于主题受益成立。

### 2.2 市场行情数据

* 用途：支持市场资金监控、市场环境判断、策略标注状态展示和数据新鲜度检查。
* 可能覆盖的模块：市场资金 / 情绪 / 宏观监控模块、重点跟踪池 / 量化策略标注模块。
* 数据可信度：中高，取决于数据供应商、延迟和授权范围。
* 是否需要人工确认：需要，尤其是数据源授权、延迟口径和异常行情。
* 风险点：不同供应商口径可能不同，延迟数据不能冒充实时数据，缺失值必须显示“数据不足”。

### 2.3 宏观数据

* 用途：支持核心利率、核心通胀、核心就业、核心GDP、原油等宏观环境监控。
* 可能覆盖的模块：市场资金 / 情绪 / 宏观监控模块、重点跟踪池 / 量化策略标注模块。
* 数据可信度：高，优先使用官方统计或权威机构数据。
* 是否需要人工确认：需要，尤其是指标口径、发布日期和修正值。
* 风险点：宏观数据常有滞后和修正，不能把单一指标直接解释为交易结论。

### 2.4 新闻 / 公告 / 政府合同数据

* 用途：捕捉新证据、新风险、政府资金流向、合同、拨款、政策事件和公司公告。
* 可能覆盖的模块：范式创新监控模块、基础投研 / 产业链图谱 / 候选股雷达模块、深度投研模块、重点跟踪池 / 量化策略标注模块、市场资金 / 情绪 / 宏观监控模块。
* 数据可信度：中到高，取决于是否来自政府文件、公司公告或可信新闻源。
* 是否需要人工确认：需要。
* 风险点：新闻摘要可能遗漏上下文，政府合同和拨款不得被伪造成真实订单或确定收入。

### 2.5 社交媒体 / KOL 线索数据

* 用途：发现线索、情绪变化、关注方向和可能的新主题。
* 可能覆盖的模块：范式创新监控模块、市场资金 / 情绪 / 宏观监控模块。
* 数据可信度：低到中，只能作为线索。
* 是否需要人工确认：必须人工确认。
* 风险点：噪声高、误导性强、传播速度快，不能直接进入事实区，不能直接生成结论。

## 3. 五个模块的数据需求

### 3.1 范式创新监控模块

需要的数据类型：

* 美国政府资金流向
* 政府合同 / 拨款
* 巨头公司科研方向
* 商界领袖公开发言
* X KOL 发言

必须说明：
KOL 信息只能作为线索，不能直接作为事实。

### 3.2 基础投研 / 产业链图谱 / 候选股雷达模块

需要的数据类型：

* 行业资料
* 产业链节点资料
* 公司业务描述
* 公司产品资料
* 证据材料
* 候选股与产业链节点绑定关系

必须说明：
候选股雷达不是买入信号。

### 3.3 深度投研模块

需要的数据类型：

* 公司主营业务
* 产品信息
* 收入分部
* 客户类型
* 财报
* 订单 / Backlog
* CapEx
* 管理层表述
* 竞争对手信息
* 风险与反证
* Claim 证据

必须说明：
没有 evidence_id 的 Claim 不能进入事实区。

### 3.4 重点跟踪池 / 量化策略标注模块

需要的数据类型：

* 主题变量
* 公司变量
* 财务变量
* 催化剂变量
* 风险变量
* 数据新鲜度
* 新证据 / 新风险
* 复查任务

必须说明：
买入价格、止盈价格、止损价格只是策略标注字段，不是交易指令。

### 3.5 市场资金 / 情绪 / 宏观监控模块

需要的数据类型：

* QQQ
* 纳斯达克指数
* 比特币
* 20日均线
* 50日均线
* 成交量
* 相对成交量
* VIX
* Put / Call
* X 社交情绪
* 核心利率
* 核心通胀
* 核心就业
* 核心GDP
* 原油

必须说明：
市场资金监控只使用 20日均线、50日均线。
禁止使用 20日涨跌幅、50日涨跌幅。
市场环境结论不是交易指令。

## 4. 数据源优先级

### P0：必须优先接入的数据

说明：
用于系统最基础的运行判断和证据链建设。优先覆盖市场环境基础数据、公司官方披露、SEC 文件、财报、evidence_items 证据库字段。

### P1：重要但可以后接的数据

说明：
用于增强深度投研、重点跟踪和复查判断。包括订单 / Backlog、CapEx、管理层表述、公司公告、行业资料、产业链节点资料。

### P2：辅助数据

说明：
用于提升解释质量和变量监控完整度。包括新闻、行业会议、政策事件、催化剂日历、竞争对手信息和数据新鲜度辅助字段。

### P3：只作为线索的数据

说明：
用于发现新方向和观察市场情绪，不能直接进入事实区。包括 X / KOL / 社交媒体线索数据。

## 5. 第一阶段建议接入顺序

1. 市场行情与市场环境基础数据
2. 公司官方披露 / 财报 / SEC 文件
3. 证据库 evidence_items
4. 深度投研 Claim 证据绑定
5. 范式创新监控数据
6. X / KOL 线索数据

必须说明：
这只是接入顺序规划，不代表已经接入真实数据。

## 6. 数据质量规则

* 每条数据必须有 data_status。
* data_status 可为：待配置、示例数据、数据不足、已验证、待人工确认。
* 每条证据必须有 evidence_id。
* 社交媒体数据必须标记为线索。
* 无法验证的数据不能进入事实区。
* 过期数据必须触发复查任务。
