# 防幻觉规则

本文件是项目防幻觉规则的唯一事实源。后续所有开发任务必须遵守。

## 1. 数据规则

1. 不得伪造真实行情数据。
2. 不得伪造真实财务数据。
3. 不得伪造真实宏观数据。
4. 不得伪造政府合同、订单、客户、CapEx、Backlog。
5. 如果没有真实数据源，必须显示“待配置”或“数据不足”。
6. 示例数据必须明确标记为“示例数据”。
7. 示例数据不得冒充真实数据。

## 2. AI 输出规则

1. AI内容必须标记为“AI草稿 / 待人工确认”。
2. AI不能直接生成最终事实结论。
3. AI不能替代人工审核。
4. AI不能把推断写成事实。
5. AI解释必须说明证据不足的部分。

## 3. 证据规则

1. 所有事实、Claim、结论必须预留 evidence_id。
2. 没有 evidence_id 的 Claim 不能进入事实区。
3. X/KOL 信息只能作为线索，不能直接作为事实。
4. KOL信息必须标记为“线索”。
5. 财报、公告、SEC文件、政府文件、公司披露优先级高于社交媒体。

## 4. 模块边界规则

1. Codex 不得自行新增未讨论模块。
2. Codex 不得自行改变模块边界。
3. 范式创新监控模块不得做选股。
4. 基础投研模块不得输出交易信号。
5. 深度投研模块不得输出买入评级。
6. 重点跟踪池模块不得自动交易。
7. 市场环境监控模块不得做个股推荐。

## 5. 交易规则

1. 本系统不提供投资建议。
2. 本系统不自动交易。
3. 本系统不自动下单。
4. 不得生成“必涨”“保证盈利”“确定收益”等内容。
5. 不得生成目标价。
6. 买入价格、止盈价格、止损价格只能作为重点跟踪池中的策略标注字段。
7. 策略标注不是交易指令。

## 6. 市场环境监控特殊规则

1. 市场资金监控必须使用 20日均线、50日均线。
2. 禁止使用 20日涨跌幅、50日涨跌幅。
3. 市场环境结论不是交易指令。
4. 市场环境模块只判断是否支持交易环境，不判断个股。

## 7. Codex 开发规则

1. 每次开发前必须读取 docs。
2. 每次开发只能完成指定任务。
3. 不得主动扩大范围。
4. 不得为了“看起来完整”而新增未讨论功能。
5. 每次修改后必须说明修改了哪些文件。
6. 如发现规则冲突，必须停止并提示用户确认。
