# evidence_items 规约

本文件说明 `data/evidence_items.example.json` 的本地证据结构。
当前阶段只创建示例结构，不接入真实 API，不生成真实数据。

## 1. evidence_items 的用途

`evidence_items` 是全系统证据库。
它用于保存五个模块引用的证据，并为事实、Claim、结论预留统一的 `evidence_id`。

任何事实性内容进入正式区域前，都必须能够追溯到 evidence_items。

## 2. 字段含义

* id：本地记录 ID。
* evidence_id：证据编号，用于连接事实、Claim、结论和 evidence_items。
* source_type：证据来源类型，例如官方披露、政府合同、新闻公告、社交媒体线索。
* source_name：证据来源名称。
* title：证据标题。
* summary：证据摘要。
* url：证据来源 URL。当前示例结构不写真实 URL。
* related_module：关联模块。
* related_ticker：关联股票占位。当前示例结构不写真实股票代码。
* related_theme：关联主题。
* evidence_level：证据等级。
* data_status：数据状态。
* clue_only：是否只能作为线索。
* human_review_required：是否需要人工确认。
* fact_allowed：是否允许进入事实区。
* created_at：创建时间。
* updated_at：更新时间。

## 3. evidence_level 的含义

* A：高可信证据，通常来自官方披露、监管文件、财报或政府文件。
* B：较可信证据，通常来自公司公告、正式材料或可验证行业资料。
* C：辅助证据，通常用于补充背景，不能单独支撑关键结论。
* D：线索级证据，通常来自社交媒体 / KOL / 未完全验证信息，不能直接进入事实区。

## 4. data_status 的含义

* 待配置：数据源或字段尚未配置。
* 示例数据：仅用于结构演示，不是真实数据。
* 数据不足：当前信息不足以支持事实判断。
* 已验证：已经完成来源核验和人工确认。
* 待人工确认：需要人工审核后才能进入正式流程。

## 5. clue_only=true 的含义

`clue_only=true` 表示该记录只能作为线索。

社交媒体 / KOL 数据必须标记为 `clue_only=true`。
线索不能直接作为事实结论，必须经过更高可信度证据验证和人工确认。

## 6. fact_allowed=false 的含义

`fact_allowed=false` 表示该记录不能进入事实区。

当证据来源是社交媒体 / KOL 线索、证据不足、未完成人工确认，或无法验证来源真实性时，应保持 `fact_allowed=false`。

## 7. 为什么没有 evidence_id 的 Claim 不能进入事实区

没有 `evidence_id` 的 Claim 无法追溯来源、无法核验、无法复查，也无法区分事实、推断、假设和待验证内容。

因此，没有 `evidence_id` 的 Claim 只能停留在草稿或待验证区域，不能进入事实区。

## 8. evidence_items 与五个模块的关系

* 范式创新监控模块：保存政府资金、科研方向、商界领袖公开发言、X / KOL 线索的证据记录。
* 基础投研 / 产业链图谱 / 候选股雷达模块：保存行业资料、产业链节点、候选股与节点绑定关系的证据记录。
* 深度投研模块：保存公司业务、产品、收入分部、客户类型、财报、订单 / Backlog、CapEx、管理层表述、竞争对手、风险与反证、Claim 验证的证据记录。
* 重点跟踪池 / 量化策略标注模块：保存主题变量、公司变量、财务变量、催化剂变量、风险变量、数据新鲜度、新证据 / 新风险和复查任务相关证据。
* 市场资金 / 情绪 / 宏观监控模块：保存市场环境、情绪、宏观指标相关证据和数据来源记录。

## 9. 当前阶段限制

* 不接 API。
* 不调用外部网络。
* 不生成真实行情数据。
* 不生成真实财务数据。
* 不生成真实宏观数据。
* 不生成真实公司结论。
* 不写真实 URL、真实公司名或真实股票代码。
* 不输出投资建议或交易建议。
* 不自动交易，不自动下单。
