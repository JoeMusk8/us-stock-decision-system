# 模块规约

本文件是项目模块边界的唯一事实源。后续 Codex 开发前必须先读取本文件。

## 1. 范式创新监控模块

定义：
范式创新监控模块是信息流监控模块，只负责发现可能值得研究的新方向。

只做：

* 四类信息源监控
* 信息流展示
* AI初筛
* AI打标签
* AI来源真伪检测
* AI行业影响分析
* 人工审核

四类信息源：

1. 美国政府资金流向
2. 巨头公司科研方向
3. 商界领袖关注方向
4. X KOL 发言

不做：

* 产业链分析
* 选股
* 深度投研
* 估值
* 交易
* 目标价

重要规则：
KOL信息只能作为线索，不能直接作为事实结论。
AI判断只是初筛，是否属于范式创新必须人工审核。

## 2. 基础投研 / 产业链图谱 / 候选股雷达模块

定义：
输入最多 3 个行业，生成产业链全景图谱，并输出最多 5 只深度投研候选股。

产业链必须拆成 6 层：

1. 上游资源 / 原材料
2. 核心零部件
3. 关键设备
4. 系统集成
5. 基础设施建设
6. 下游应用 / 客户

瓶颈识别必须使用 10 个维度：

1. 技术难度
2. 供给稀缺
3. 产能瓶颈
4. 客户认证周期
5. 资本开支强度
6. 替代难度
7. 毛利率潜力
8. 议价能力
9. 交付周期
10. 监管约束

输出：
最多 5 只候选股票。

重要规则：
候选股雷达不是买入信号，只代表该股票值得进入深度投研。
没有证据ID的节点和股票，不能进入正式候选。

## 3. 深度投研模块

定义：
对“股票 + 主题”进行证据驱动的公司级验证，判断是否进入重点跟踪池。

最多支持手动填写 15 只股票。

必须验证的问题：

1. 公司主营业务是什么？
2. 主要产品是什么？
3. 收入分部是什么？
4. 客户类型是什么？
5. 该主题相关业务是否属于核心业务？
6. 该业务是否只是边缘业务？
7. 公司是否明确披露过该方向？
8. 这家公司是否真的受益于这个主题？

研究等级：
S：进入核心跟踪池
A：进入重点跟踪池
B：进入观察池
C：证据不足，继续观察
D：剔除

重要规则：
研究等级不是买入评级，不代表交易建议。
没有 evidence_id 的 Claim 不能进入事实区。

## 4. 重点跟踪池 / 量化策略标注模块

定义：
保存高质量标的，持续跟踪核心变量，接收新证据和新风险，提醒重新研究，并进行策略价格标注。

跟踪变量：

1. 主题变量
2. 公司变量
3. 财务变量
4. 催化剂变量
5. 风险变量
6. 数据新鲜度变量

量化策略标注：
本项目不单独设置量化交易决策模块。
量化策略标注合并在重点跟踪池中。

只允许标注：

* 买入价格
* 止盈价格
* 止损价格

重要规则：
买入价格、止盈价格、止损价格只是策略标注字段，不是交易指令。
不自动交易。
不自动下单。

## 5. 市场资金 / 情绪 / 宏观监控模块

定义：
完全独立的市场环境风控模块，只判断当前市场环境是否支持交易。

拆成三部分：

1. 市场资金监控
2. 市场情绪监控
3. 宏观环境监控

市场资金监控：
监控 QQQ、纳斯达克指数、比特币。

必须使用：

* 当前价
* 20日均线
* 50日均线
* 20日均线方向
* 50日均线方向
* 成交量
* 相对成交量
* 是否强于基准
* 是否出现风险偏好扩散

禁止使用：

* 20日涨跌幅
* 50日涨跌幅

市场情绪监控：

* VIX
* Put / Call
* X 社交情绪

宏观环境监控：

* 利率：核心利率
* 通胀：核心通胀
* 就业：核心就业
* GDP：核心GDP
* 能源：原油

重要规则：
市场环境监控模块不是选股模块。
市场环境结论不是交易指令，只是辅助风控判断。
