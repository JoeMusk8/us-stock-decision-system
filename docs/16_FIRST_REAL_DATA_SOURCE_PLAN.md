# Phase 4A 第一个真实低风险数据源接入规划

## 1. 本阶段结论

本阶段只规划第一个真实低风险数据源，不接入 API，不调用外部网络，不生成真实数据。

建议第一个真实数据源选择：

市场环境基础行情数据

优先覆盖：

* QQQ
* 纳斯达克指数
* 比特币
* VIX

暂不覆盖：

* 个股财报
* 公司 Claim
* 政府合同
* X / KOL 数据
* 自动交易数据
* 订单 / Backlog
* SEC 文件解析

## 2. 为什么先接市场环境基础行情数据

* 它服务于市场资金 / 情绪 / 宏观监控模块。
* 它不直接产生个股推荐。
* 它不直接产生买卖建议。
* 它不进入公司事实区。
* 它适合先验证数据接入、字段标准化、data_status、错误处理和 UI 展示流程。
* 它比公司财报、政府合同、X/KOL 数据更低风险。

## 3. 第一个数据源候选方案

以下内容只是候选方案，不代表已经接入。

候选名称：
market_environment_basic_data

候选覆盖资产：

* QQQ
* 纳斯达克指数
* 比特币
* VIX

候选字段：

* symbol
* asset_name
* asset_type
* current_price
* ma_20
* ma_50
* ma_20_direction
* ma_50_direction
* volume
* relative_volume
* data_status
* source_name
* source_type
* updated_at

特别规则：

* 市场资金监控只允许使用 20日均线、50日均线。
* 禁止使用 20日涨跌幅。
* 禁止使用 50日涨跌幅。
* 市场环境结论不是交易指令。
* 本系统不自动交易，不自动下单。

## 4. 数据状态规则

刚接入时，数据状态不得直接标记为“已验证”。

允许状态：

* 待配置
* 数据不足
* 待人工确认
* 已验证

首次接入后建议流程：

1. 拉取数据
2. 标记为“待人工确认”
3. 进行字段检查
4. 进行异常值检查
5. 人工确认后才允许标记为“已验证”

## 5. 错误处理要求

必须规划以下错误场景：

* 数据源不可用：显示“数据不足”
* 网络失败：显示“数据不足”
* 返回空数据：显示“数据不足”
* 字段缺失：显示“数据不足”
* 日期异常：显示“数据不足”
* 数值异常：显示“数据不足”
* 价格为 0：显示“数据不足”
* 均线无法计算：显示“数据不足”
* 成交量缺失：显示“数据不足”
* 比特币数据源与股票数据源不一致：显示“数据不足”
* VIX 数据源不可用：显示“数据不足”

不得用假数据填充。

## 6. 安全边界

* 不得因为行情数据上涨就输出买入建议。
* 不得因为行情数据下跌就输出卖出建议。
* 不得生成目标价。
* 不得生成保证盈利结论。
* 不得自动交易。
* 不得自动下单。
* 市场环境模块只判断环境，不判断个股。
