# 验收标准

本文件是项目验收标准的唯一事实源。

## 1. 全局验收标准

1. 项目必须可以运行。
2. 不得出现破损 import。
3. 不得出现未讨论模块。
4. 不得出现自动交易功能。
5. 不得出现自动下单功能。
6. 不得出现投资建议。
7. 不得出现伪造真实数据。
8. 所有示例数据必须标记为“示例数据”。
9. 数据不足必须显示“数据不足”。
10. AI内容必须显示“AI草稿 / 待人工确认”。

## 2. 模块数量验收

系统必须包含 5 个模块：

1. 范式创新监控模块
2. 基础投研 / 产业链图谱 / 候选股雷达模块
3. 深度投研模块
4. 重点跟踪池 / 量化策略标注模块
5. 市场资金 / 情绪 / 宏观监控模块

不得新增第 6 个模块。

## 3. UI 页面数量验收

范式创新监控模块：4 页
基础投研模块：6 页
深度投研模块：11 页
重点跟踪池模块：10 页
市场环境监控模块：6 页

## 4. 模块边界验收

1. 范式创新监控模块只做信息流监控和AI初筛。
2. 基础投研模块最多输入 3 个行业，最多输出 5 只候选股。
3. 候选股雷达不是交易信号。
4. 深度投研模块最多支持手动填写 15 只股票。
5. 深度投研必须绑定“股票 + 主题”。
6. 重点跟踪池包含量化策略标注，但不自动交易。
7. 不单独生成量化交易模块。
8. 市场环境监控模块是独立风控模块，不做选股。

## 5. 数据结构验收

必须保留 11 张数据表：

1. evidence_items
2. paradigm_items
3. industries
4. industry_nodes
5. candidate_stocks
6. research_tasks
7. claims
8. tracking_items
9. tracking_variables
10. market_indicators
11. review_tasks

## 6. 市场环境验收

1. 市场资金监控必须包含 QQQ、纳斯达克指数、比特币。
2. 必须使用 20日均线、50日均线。
3. 不得使用 20日涨跌幅、50日涨跌幅。
4. 市场情绪必须包含 VIX、Put / Call、X 社交情绪。
5. 宏观环境必须包含核心利率、核心通胀、核心就业、核心GDP、原油。

## 7. 重点跟踪池验收

1. 必须包含主题变量、公司变量、财务变量、催化剂变量、风险变量、数据新鲜度。
2. 买入价格、止盈价格、止损价格只能作为策略标注字段。
3. 必须明确提示：不自动交易，不自动下单。
